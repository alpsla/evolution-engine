## 1. Introduction

This Privacy Policy describes how CodeQual ("we," "us," or "our") collects, uses, shares, and protects personal information when you use the Evolution Engine software, the codequal.dev website, and related services (collectively, the "Service").

Evolution Engine is a local-first command-line interface (CLI) tool for software development analytics. The core analysis pipeline runs entirely on your machine. Your source code never leaves your computer. Data is transmitted to our servers or third-party services only when you explicitly initiate specific features, as described in this policy.

We are committed to protecting your privacy and being transparent about our data practices. Please read this policy carefully. By using the Service, you acknowledge that you have read and understood this policy.

## 2. What Data We Collect

### 2.1 Data That Stays on Your Machine (Never Transmitted)

The following data is created and stored locally on your computer in the `~/.evo/` and `<repository>/.evo/` directories. We never receive, access, or transmit this data:

- **Pipeline output:** Events, signals, explanations, patterns, and advisories generated by Phases 1 through 5
- **Configuration files:** Your preferences stored in `~/.evo/config.toml`
- **GitHub API cache:** Cached responses from the GitHub API stored in `.evo/api_cache/`
- **Investigation reports:** Results from AI-powered investigation and fix operations
- **HTML reports:** Generated analysis reports

Git commit metadata (which may include committer names and emails from your local git log) is read from your local repository and stored only in local `.evo/` directories. This data is never transmitted to us.

### 2.2 CLI Telemetry (Opt-In)

When you opt in to telemetry, the following anonymous usage data is collected:

- **Event name** (e.g., "analyze_complete")
- **Aggregate counts** (adapter count, signal count, pattern count)
- **Family list** (categories of data sources used, e.g., "version_control," "dependency")
- **CLI version number**
- **Anonymous identifier** (a randomly generated UUID4, stored at `~/.evo/anon_id`, not derived from any personal information)

**What is NOT collected by telemetry:** File paths, source code, repository names, usernames, commit hashes, or IP addresses.

Telemetry is **disabled by default**. It is enabled only if you explicitly opt in via an interactive prompt or by running `evo telemetry on`. You can disable it at any time by running `evo telemetry off`, setting the `DO_NOT_TRACK=1` environment variable, or running `evo config set telemetry.enabled false`.

### 2.3 Payment Information (Stripe)

When you purchase a Pro subscription, payment is processed by Stripe, Inc. You enter your payment information directly on Stripe's hosted checkout page. We never see, handle, or store your credit card number, CVV, or billing address.

From Stripe, we receive and process:

- **Email address** (associated with your Stripe account) — this is used only to generate the hashed identifier in your license key and is not stored in our systems beyond Stripe's retention
- **Stripe customer ID** (a pseudonymous identifier assigned by Stripe)
- **Subscription status** (active, canceled, etc.)

We never log or retain your plaintext email address.

### 2.4 License Key

Your license key contains:

- **Subscription tier** (Example: "pro")
- **A truncated cryptographic hash** (first 16 characters of SHA-256) of the email address associated with your Stripe purchase
- **Issuance date**

The license key is stored locally on your machine at `~/.evo/license.json` or in the `EVO_LICENSE_KEY` environment variable. The key is base64-encoded and HMAC-signed, but not encrypted. The email hash is irreversible — the original email address cannot be recovered from the key. This design ensures that even if the license key is exposed, your email address remains protected.

We use the email address provided by Stripe only to generate this hashed identifier. The plaintext email is not stored in any Evolution Engine database or log; it remains with Stripe subject to their privacy policy and retention practices.

### 2.5 Adapter Request Form

If you submit a request for a new adapter integration through our website, we collect:

- **Email address**
- **Adapter name and family** (e.g., "Jenkins," "ci")
- **Description and use case** (free-text fields you provide)
- **IP address** (used only for rate limiting; not stored or logged)

Please note: if a GitHub bot token is configured, your adapter request (including your email address) may be posted as a public GitHub issue. You will be informed of this before submission.

### 2.6 Axiom Operational Logs

Our server-side handlers log operational data to Axiom, a log aggregation service. The following is logged:

| Source | Data Logged | Contains Personal Data |
|--------|-------------|:----------------------:|
| Telemetry handler | Event name, aggregate counts, anonymous ID, version, timestamp | No |
| Stripe webhook | Truncated hash of Stripe customer ID, subscription tier, timestamp | No (hashed, irreversible) |
| Adapter request handler | Adapter name, family, description, use case, email, timestamp | Yes (email) |
| Error logs | Event type, error class name, timestamp | No (sanitized — no exception details) |

### 2.7 AI Investigation (User-Initiated)

When you run `evo investigate` or `evo fix`, and have configured an AI provider API key, the following data is sent to the AI provider (e.g., Anthropic):

- **Repository directory name** (not the full path)
- **Analysis period dates**
- **Metric deviations** (family names, metric names, numerical values)
- **Pattern match descriptions** (generic statistical text)

**What is NOT sent:** Source code, file contents, git diffs, commit messages, author names, or file paths.

This feature requires you to explicitly provide an API key and invoke the command. No AI API calls are made unless you initiate them.

### 2.8 KB Sync (Community Pattern Sharing, Opt-In)

Knowledge Base synchronization is **disabled by default** (privacy level 0). If you opt in:

- **Level 1:** Only aggregate metadata is shared (family counts, change counts, pattern match counts, timestamp).
- **Level 2:** Anonymized pattern digests are shared (fingerprint hashes, family names, metric names, correlation strength, occurrence count).

No repository names, source code, file paths, usernames, or other identifying information is included at any privacy level.

### 2.9 GitHub Integration (PR Comments and Inline Suggestions)

When the Evolution Engine GitHub Action posts PR comments or inline suggestions, the following is sent to the GitHub API:

- Risk level indicators
- Family and metric names with numerical values
- Pattern match descriptions
- AI investigation summaries (if investigation was run)
- File paths and line numbers (for inline suggestions, within the repository only)

No personal data is included in PR comments or inline suggestions.

## 9. Data Security

We implement the following security measures to protect your data:

- **Local-first architecture:** The core analysis pipeline runs entirely on your machine. Source code never leaves your computer.
- **HMAC-signed license keys:** License keys are cryptographically signed to prevent tampering.
- **Stripe webhook signature verification:** All incoming Stripe webhooks are verified using Stripe's signature mechanism to prevent spoofing.
- **Rate limiting:** API endpoints implement rate limiting to prevent abuse.
- **Payload validation:** All incoming data is validated for size, type, and content before processing.
- **KB security validation:** Community patterns imported via KB sync are validated for injection attacks, oversized payloads, fingerprint spoofing, and type confusion.
- **Minimal data collection:** We collect only the data necessary for each feature to function.
- **Encryption in transit:** All data transmitted between your machine and our servers uses HTTPS/TLS encryption.
- **Access controls:** Server-side secrets (API keys, tokens) are stored as environment variables with restricted access.
- **No persistent server-side database:** We do not maintain a database of user accounts or personal data. Payment data is managed by Stripe, and logs are managed by Axiom.

While we take reasonable measures to protect your data, no method of transmission over the internet or electronic storage is completely secure. We cannot guarantee absolute security.
